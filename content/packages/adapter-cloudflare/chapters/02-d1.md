# D1 戦略

```ts
import { createD1Adapter } from '@f12o/papyr-adapter-cloudflare';

const adapter = createD1Adapter({
  db: env.PAPYR_DB,
});
```

`createD1Adapter` は `papyr_documents` table を既定で使い、`id` と `doc` の 2 列に document を保存します。package 側で migration は作らないので、table は先に用意しておきます。

```sql
CREATE TABLE papyr_documents (
  id TEXT PRIMARY KEY,
  doc TEXT NOT NULL
);
```

`put` は `ON CONFLICT(id) DO UPDATE` で upsert し、`get` / `delete` は単純な primary key lookup です。KV より一覧や絞り込みを拡張しやすく、将来的に metadata 列や独自 index を足したいときのベースに向いています。
