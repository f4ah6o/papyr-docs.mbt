# R2 戦略

```ts
import { createR2Adapter } from '@f12o/papyr-adapter-cloudflare';

const adapter = createR2Adapter({
  bucket: env.PAPYR_DOCS_R2,
  prefix: 'papyr-docs/',
});
```

R2 adapter は document を `${prefix}${id}.json` で保存します。`list` は prefix 配下の object を列挙してから各 JSON を読み直す構成なので、**静的配布寄りの read-heavy なケース** と相性が良いです。

この docs site も manifest + per-document JSON を R2 に置く構成で、公開用データ配布に R2 を使っています。KV のような key-value の単純さと、object storage の扱いやすさの中間を取りたいときに向いています。
